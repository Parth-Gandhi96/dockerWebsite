Virtualization

Virtualization is the act of using abstraction to make software look and behave like hardware, with corresponding benefits in flexibility, cost, scalability, reliability, and often overall capability and performance in a broad range of applications. This practice allows IT organizations to operate multiple operating systems, more than one virtual system, and other applications on a single server. OS virtualization is allowing software to use a piece of hardware to run multiple operating system images at the same time.

For example, you have 3 physical servers with individual dedicated purposes. Server 1 is a mail server, server 2 a web, and server 3 runs internal legacy applications. Each server uses about 30% capacity which is only a fraction of their running potential.

With the use of virtualization we now have the ability to split the mail server into 2 unique ones that can handle independent tasks so that the legacy apps can be migrated. Furthermore, you could split the first server again so that it can handle another task which increases its use from 30% to 60% to 90%. Once the server is migrated you can use the now empty server for other tasks or retire it altogether which can be more cost effective by reducing maintenance costs.

How does virtualization work?

A software called hypervisors separate the physical resource from the virtual environments. Hypervisors can either sit on top of an operating system, like a laptop, or can be installed directly onto the hardware, like a server. Hypervisors take the physical resource and divide them up so that virtual environments can use them. The resources are portioned as needed and users interact within the virtual environment, this is typically called a guest machine or virtual machine.

The virtual machine functions as a single data file and can be moved from one computer to another, opened in either one, and be expected to work the same, similar to a digital file. When the virtual environment is running and a user or program needs resources from the physical environment the hypervisor sends a request to the physical system which then cashes and changes.

Types of virtualization:

  • Data Virtualization
  • Desktop Virtualization
  • Server Virtualization
  • Operating System Virtualization
  • Network Functions Virtualization
