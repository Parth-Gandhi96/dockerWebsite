Linux Basic Commands

Command Description
pwd

To find out the path of the current working directory
cd

To navigate through the Linux files and directories

cd .. (with two dots) to move one directory up cd to go straight to the home folder cd- (with a hyphen) to move to your previous directory

ls

To view the contents of a directory

ls -R will list all the files in the sub-directories as well

ls -a will show the hidden files

ls -al will list the files and directories with detailed information like the permissions, size, owner, etc.

cat

To list the contents of a file on the standard output

cat > filename creates a new file

cat filename1 filename2>filename3 joins two files (1 and 2) and stores the output of them in a new file (3) to convert a file to upper or lower case use, cat filename | tr a-z A-Z >output.txt

cp

To copy files from the current directory to a different directory.

cp scenery.jpg /home/username/Pictures

mv

To move files, although it can also be used to rename files.

mv file.txt /home/username/Documents

mkdir

To make a new directory

rm

To delete directories and the contents within them (rm -r → for recursive delete in directory)

touch

To create a blank new file

locate

To locate a file, just like the search command in Windows [Similar cmd: find]

grep

Lets you search through all the text in a given file

sudo

Short for “SuperUser Do”, this command enables you to perform tasks that require administrative or root permissions.

df

To get a report on the system’s disk space usage, shown in percentage and KBs

du

To check how much space a file or a directory takes

head

To view the first lines of any text file

tail

Similar function to the head command, but instead of showing the first lines, the tail command will display the last ten lines of a text file.
diff

Compares the contents of two files line by line
tar

Most used command to archive multiple files

chmod

Used to change the read, write, and execute permissions of files and directories

How to log into your docker container

Login to a self-hosted registry

If you want to login to a self-hosted registry you can specify this by adding the server name.

$ docker login localhost:8080

Provide a password using STDIN

To run the docker login command non-interactively, you can set the --password-stdin flag to provide a password through STDIN. Using STDIN prevents the password from ending up in the shell’s history, or log-files.

The following example reads a password from a file, and passes it to the docker login command using STDIN:

$ cat ~/my_password.txt | docker login --username foo --password-stdin
